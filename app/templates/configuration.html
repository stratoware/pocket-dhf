Configuration Notes
  • Unique IDs: Each option has a permanent ID (S1, S2, S3 for severity; P1, P2, P3 for probability) that never changes.
  • Display Names: The names shown to users can be modified without affecting existing data compatibility.
  • Adding Options: New options get the next available ID automatically (e.g., S4, P4, etc.).
  • Removing Options: Options currently in use by existing risks cannot be removed (shown with "In Use" badge).
  • Descriptions: Optional descriptions help users understand when to use each option.
  • Data Persistence: All changes are saved automatically and tracked in version control.
  • Backwards Compatibility: The ID system ensures existing data remains valid even when display names change.
